Hexo + Github Page 建置個人 blog - (3) hexo 指令

發表於2022-10-14|更新於2022-10-14
|閱讀量:

前情提要

前面我們已經把 Hexo 跟 Github Page 串接好了，接下來我們要來把資料丟上去 Github Page 還有創建各種各式各樣的個人化咚咚ㄌ。

我相信一定會有人問說為什麼不在上一篇就把 push 的部分寫好，但我覺得這些東西都是 hexo 的指令，所以我把它全部放在一篇，這樣以後要查詢也比較好查。

Push 上去 Github Page

實際上很簡單，只要一行指令就解決。

輸入之後應該就能看到 github 上出現檔案。

創建文章

  • 輸入後應該會跳出這樣子

完成後就能在 source/_post 的資料夾裡面找到你剛剛創的文章

hexo 的文章都是用 md 格式存檔，所以撰寫的時候可以直接用 Markdown 語法寫。

  • 打開文章之後它會有這些預設內容

    • title: 名稱
    • date: 文章日期
    • tags: 你的 tag

記住這三行可以修改，但不要刪除。

後續指令會隨時更新，但主要就這兩個指令在跑而已
