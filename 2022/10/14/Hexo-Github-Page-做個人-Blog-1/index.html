Hexo + Github Page 建置個人 Blog - (1) 建立 Hexo 環境

發表於2022-10-14|更新於2022-10-14
|閱讀量:

前情提要

為了建置一個自己的 Blog 來講幹話，所以我找了許許多多各式各樣奇奇怪怪的框架、教學，但這些東西不外乎就是需要各式語言、前置知識，這讓我很難受，所以我就不想學了。

然後在某個瞬間我遇到了 Hexo 這個酷酷的靜態網頁框架，而且他可以透過指令把你更新的東西丟到 Github Page 上面。

你不用學 JS、Golang、PHP 等前端後端語言，你只要會打字好像就能架惹。

開始安裝一些奇奇怪怪的東西

  • Node

    • 官網下載，他出現哪個寫推薦就下載哪個

      無腦按下一步就好，完成後打開 cmd，輸入 node –version 有跳出版本就是安裝成功

    • 我用 16 的版本

  • Hexo

    • 下載完以後在 cmd 輸入 npm install -g hexo-cli
    • 等他跑完之後輸入 hexo –version
    • 沒有報錯就是完成ㄌ
    • 如果出現 hexo: commend not foud，請參考此連結設定系統路徑

  • Git

    • 官網下載，一樣無腦安裝就好

  • VSCode

    • 官網下載，這應該不用教，對吧(?)

開始建置 Hexo 環境

  1. 在你想布置的地方輸入

    • 安裝完成會長這樣

    • 如果跳出這個東西，將路徑改成其他地方就沒事ㄌ

  2. cd (名稱)，進去安裝依賴套件

    • 輸入 npm init 後，他會跳出一堆東西，全部按 Enter 就好

    • 然後看到這個畫面輸入 yes

    • 打開你安裝的資料夾應該會出現這堆東西

      node_modules：依賴套件
      scaffolds：文章模板
      source：文章存放位置
      themes：主題
      .npmignore：丟上去的時候需要忽略的文件，這個可以不管她
      _config.landscape.yml：主題配置的文件
      _config.yml：頁面的配置文件
      package.json：專案名稱、版本、描述之類的訊息

  3. 輸入 hexo server 打開伺服器

  4. 打開瀏覽器連線 http://localhost:4000/ 能看到現在的樣子，有出現就代表成功ㄌ
